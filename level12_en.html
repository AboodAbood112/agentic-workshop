🔀 Level 12 merges the two tracks:
An ADW (Python) manages the Orchestrator, which manages teams of non-deterministic Agents.
Code reliability + Agent creativity = True autonomy

2 The Three-Layer Architecture

Layer 1: ADW (Python) — Deterministic Control
Validates results, logs checkpoints, retries, manages checkpoints
Promises: Verified results meeting acceptance criteria
↕
Layer 2: Orchestrator (Agent) — Non-Deterministic Execution
Receives task from ADW, delegates: Scouts → Planner → Builder → Reviewer
Promises: Workflow completion with a structured status report
↕
Layer 3: Sub-Agents — The Specialists
Scout searches | Planner designs | Builder executes | Reviewer validates
They promise: Structured output in a known format

4 Workflow Types

plan_build Scout → Plan → Build For simple features
plan_build_review Scout → Plan → Build → Review Standard workflow
plan_build_review_fix Scout → Plan → Build → Review → Fix (Loop) For complex features

⚠️ Common Mistake

Setting MAX_FIX_CYCLES too high. 2-3 attempts are enough. If Builder doesn't fix it in 3 tries, the problem is in the plan, not execution — escalate to a human.

5 Practical Exercise Steps

1

Parses 3 arguments (ADW name, workflow type, prompt). Defines phases for each workflow type. Creates tracking records in adws/active/. Includes MAX_FIX_CYCLES = 2.

Read-only. Reads adws/active/{name}.json + adws/logs/{name}.log. Displays current phase, progress list, latest log entries. Alerts if ADW is stalled for +5 mins. Lightweight model.

7 Key Takeaways

🤝Deterministic + Non-Deterministic = Reliable Autonomy — Python controls the when and why. Agents control the how.
🔢Trust = Determinism × Observability × Validation × Recovery — Every factor is mandatory.
🚧Validation Gates Catch Failures Early — ADW verifies before advancing to each phase.
⏱️Recovery Budget Prevents Infinite Loops — 2-3 retries then escalate to human.
📐Hierarchy Bounds Autonomy at Every Layer — Each layer constrains the one below it.
