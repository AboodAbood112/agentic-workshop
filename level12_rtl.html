Class 3, Gear 3 — الشكل النهائي

"الشكل النهائي: كود حتمي يُنسّق Agents غير حتمية عبر Orchestrator. أقصى استقلالية. أقصى ثقة."

١ دمج المسارين

🌀 Track 1: غير حتمي (L1-L8)
Agents مدفوعة بالـ Prompts
إبداعية وتكيفية
لكن: ناتج متغير
تطلب من النموذج — لا تأمره
⚙️ Track 2: حتمي (L9)
Python يتبع خطوات دقيقة
يتحقق في كل نقطة
لكن: Agent واحد في كل مرة
تأمر النظام — لا تطلب
🔀 Level 12 يدمج المسارين:
ADW (Python) يُدير الـ Orchestrator، الذي يُدير فرق Agents غير حتمية.
موثوقية الكود + إبداع الـ Agents = استقلالية حقيقية

٢ المعمارية الثلاثية الطبقات

طبقة 1: ADW (Python) — التحكم الحتمي
يتحقق من النتائج، يُسجّل الـ checkpoints، يُعيد المحاولة، يُدير نقاط التحقق
يعد بـ: نتائج متحقق منها تلبي معايير القبول
↕
طبقة 2: Orchestrator (Agent) — التنفيذ غير الحتمي
يستقبل المهمة من ADW، يُفوّض: Scouts → Planner → Builder → Reviewer
يعد بـ: اكتمال الـ Workflow مع تقرير حالة منظم
↕
طبقة 3: Sub-Agents — المتخصصون
Scout يبحث | Planner يصمم | Builder ينفّذ | Reviewer يتحقق
يعدون بـ: ناتج منظم بشكل معروف

أزل أي عامل واحد → النظام يصبح غير موثوق

٤ أنواع الـ Workflow

plan_build Scout → Plan → Build للميزات البسيطة
plan_build_review Scout → Plan → Build → Review سير العمل المعياري
plan_build_review_fix Scout → Plan → Build → Review → Fix (حلقة) للميزات المعقدة

⚠️ خطأ شائع

وضع MAX_FIX_CYCLES عالياً جداً. 2-3 محاولات كافية. إذا لم يُصلح Builder في 3 مرات، المشكلة في الخطة لا التنفيذ — صعّد للإنسان.

٥ خطوات التمرين العملي

١
أنشئ orch_start_adw.md

يُحلّل 3 arguments (اسم ADW، نوع الـ workflow، الـ prompt). يُعرّف المراحل لكل نوع workflow. ينشئ tracking records في adws/active/. يُضمّن MAX_FIX_CYCLES = 2.

٢
أنشئ orch_check_adw.md

للقراءة فقط. يقرأ adws/active/{name}.json + adws/logs/{name}.log. يُظهر المرحلة الحالية، قائمة التقدم، آخر إدخالات السجل. يُنبّه إذا كان ADW متوقفاً +5 دقائق. نموذج خفيف.

٣
أنشئ adw_with_orchestrator.py

3 كلاسات منفصلة:

  • OrchestratorClient — يُغلّف استدعاء pi CLI
  • ADWRecord — dataclass لتتبع الحالة
  • ADWEngine — منطق الـ workflow مع بوابات التحقق
python adw_with_orchestrator.py "Add task priority" --workflow plan_build_review
٤
اكتب SYSTEM_ARCHITECTURE.md

خريطة كاملة للـ 12 Levels — 5 طبقات: Foundation (L1-3)، Quality (L4-5)، Composition (L6-7)، Autonomy (L8-9)، Orchestration (L10-12). مع تدفق البيانات ونموذج الثقة.

python adws/adw_orchestrated/adw_with_orchestrator.py \
    "Add task priority field" \
    --workflow plan_build_review

٧ الدروس المستفادة

🤝حتمي + غير حتمي = استقلالية موثوقة — Python يتحكم بالـ متى والـ لماذا. الـ Agents يتحكمون بالـ كيف.
🔢الثقة = Determinism × Observability × Validation × Recovery — كل عامل إلزامي.
🚧بوابات التحقق تمسك الفشل مبكراً — ADW يتحقق قبل الانتقال لكل مرحلة.
⏱️ميزانية الإصلاح تمنع الحلقات اللانهائية — 2-3 محاولات ثم التصعيد للإنسان.
📐التسلسل الهرمي يحدود الاستقلالية في كل طبقة — كل طبقة تُقيّد التي تحتها.
