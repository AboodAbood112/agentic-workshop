🤖 Level 1: Prime Prompts & Memory Files

Class 1, Gear 1 — The foundation of every Agentic layer

"If you don't tell the Agent who it is, what it's working on, and how to work — it will guess. And it will guess wrong."

1 What is AGENTS.md?

It is the Prime Prompt file — it lives in the root of the project and loads automatically every time the Agent starts a new session.

Think of it as a briefing document you'd give to a new engineer on their first day:

  • What is this project?
  • What tools do we use?
  • How do I run things?
  • What are the conventions?
  • What things should I pay attention to?

📍 Where does the file live?

LocationScope
./AGENTS.mdProject Root — for all sessions
~/.pi/agent/AGENTS.mdGlobal — for all projects
./apps/backend/AGENTS.mdSpecific directory — for monorepos

2 The Goldilocks Zone

❌ Too short — "Figure It Out"

# TaskFlow
Uses bun. Run `bun dev` to start.

Problem: The Agent will guess everything and make mistakes — file structures, test patterns, schemas...

❌ Too long — "Encyclopedia"

A 180+ line file detailing every spelling rule and ESLint configuration...

Problem: Important information gets lost in the noise and context window is wasted.

✅ The Perfect Size — 30 to 50 lines

#SectionContent
1Context2-3 sentences about the project
2ToolingRuntime, frameworks, libraries
3Key CommandsInstall, dev, test, build
4Project StructureMain directory map
5Dev Guidelines3-7 rules that actually matter
6Important NotesThings that cause issues, special patterns

3 Practical Exercise — TaskFlow

The project is TaskFlow — a task management application. You are required to:

Steps:

  • Explore the project structure in workshop/app/
  • Understand the tools and commands from package.json
  • Discover conventions (Zod schemas, status and priority values)
  • Write AGENTS.md in 30-50 lines
  • Compare your version with the files in completed/

🔍 Checking your version:

cat workshop/app/AGENTS.md
wc -l workshop/app/AGENTS.md        # Expected: 30-50 lines
grep -c "bun" workshop/app/AGENTS.md
grep -c "Hono\|hono" workshop/app/AGENTS.md
grep -c "SQLite\|sqlite" workshop/app/AGENTS.md

4 Key Takeaways

🏗️AGENTS.md is the foundation — every other technique in the workshop builds on top of it.
🎯Be precise with commands — bun run --filter @taskflow/backend dev is better than "run the backend".
✂️Include what matters, remove what doesn't — every line must earn its place.
🔄Update it as the project evolves — an outdated file is worse than no file.
📓Memory files — keep permanent facts in AGENTS.md, not in the chat history.
💡 AGENTS.md = The Agent's Memory
Without it, every session starts from scratch and guesses.
With it, it starts like an engineer who knows the project from day one. 🚀
