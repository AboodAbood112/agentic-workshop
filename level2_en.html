🤖 Level 2: Sub-Agents, Plans & AI Docs

Class 1, Gear 2 — Empowering the Agent with specialists, knowledge, and workflows

"A single Agent trying to do everything is like a single engineer trying to be the Architect, Frontend, Backend, and QA all at once."

1 What are Sub-Agents?

They are specialized prompt files that live in .pi/prompts/agents/ — each one defines a specialized expert with a specific goal, limited tools, and clear steps.

📋 Sub-Agent File Format:

---
name: agent-name
description: Action-oriented description. Use proactively when...
tools: Read, Write, Bash
model: sonnet
---

# Purpose
You are a specialized [role]. Your job is to [primary task].

## Instructions
1. Step one
2. Step two

## Output Format
Describe what the agent produces.

⚠️ Common Mistake

Giving sub-agents too many tools. Start with the minimum tools possible.

🗂️ Examples of Sub-Agents:

AgentGoalTools
meta-agentCreates new agentsWrite, WebFetch
docs-scraperFetches and summarizes docsWebFetch, Write, Read, Glob
code-reviewerReviews code qualityRead, Grep, Glob
test-writerGenerates test filesRead, Write, Bash

2 What are AI Docs?

They are specialized knowledge files stored in ai_docs/ — they give the Agent a deep understanding of the system.

  • AGENTS.md → Tells the Agent what the project is and how to run it
  • AI Docs → Tells it how the system works internally and why it was built this way

📁 Typical Files:

FileContent
architecture.mdOverview, data flow, technical decisions
api-reference.mdEvery endpoint documented with request and response
component-guide.mdDesign system components with props and variants

⚠️ Common Mistake

Cramming AI Docs content inside AGENTS.md. AGENTS.md should be 30-50 lines only. Keep them separate.

3 What are Plan Commands?

They are reusable workflows stored in .pi/prompts/ — invoked with / like /prime or /quick-plan.

📋 Command File Format:

---
description: What this command does
argument-hint: <optional description>
---

# Command Name
Instructions for the agent to follow.
Use $ARGUMENTS to reference user input.

Example: /quick-plan add a priority filter to the task list — everything after the command name becomes $ARGUMENTS

4 Practical Exercise Steps

1
Create the meta-agent

Create .pi/prompts/agents/meta-agent.md — its only job is to create other agents.

2
Create the docs-scraper agent

Create .pi/prompts/agents/docs-scraper.md — it fetches documentation from URLs and saves structured summaries.

3
Create AI Docs for TaskFlow

Create 3 files in ai_docs/: architecture.md, api-reference.md, component-guide.md

4
Create Plan Commands

Create 3 commands in .pi/prompts/: prime.md, quick-plan.md, load_ai_docs.md

5
Test the Commands

Try /prime and /quick-plan add a search feature to TaskFlow — does it give organized and useful results?

5 Key Takeaways

🧑‍💼Sub-agents = Specialists — each one does one thing excellently with minimal tools.
📚AI Docs = Knowledge Base — tells the Agent how the system works, not just how to run it.
⚡Commands = Ready Workflows — turns multiple steps into a single command that produces consistent results.
🔀Together they enable parallel work — multiple agents working on different things simultaneously.
🗂️Separation of Concerns — AGENTS.md for context, ai_docs for knowledge, commands for workflows, agents for specialization.
💡 Difference between Level 1 and Level 2
Level 1: You gave the Agent a briefing document 📄
Level 2: You gave it a specialized team + knowledge base + standard procedures 🚀
