🤖 Level 2: Sub-Agents, Plans & AI Docs

Class 1, Gear 2 — إعطاء الـ Agent متخصصين، معرفة، وسير عمل

"Agent واحد يحاول يعمل كل شيء، مثل مهندس واحد يحاول يكون المعماري والـ Frontend والـ Backend والـ QA في نفس الوقت."

١ ما هي الـ Sub-Agents؟

هي ملفات prompt متخصصة تعيش في .pi/prompts/agents/ — كل واحدة بتعرّف خبير متخصص بهدف محدد، أدوات محدودة، وخطوات واضحة.

📋 شكل ملف الـ Sub-Agent:

---
name: agent-name
description: Action-oriented description. Use proactively when...
tools: Read, Write, Bash
model: sonnet
---

# Purpose
You are a specialized [role]. Your job is to [primary task].

## Instructions
1. Step one
2. Step two

## Output Format
Describe what the agent produces.

⚠️ خطأ شائع

إعطاء الـ sub-agents أدوات أكثر من اللازم. ابدأ بأقل الأدوات الممكنة.

🗂️ أمثلة على Sub-Agents:

الـ Agentالهدفالأدوات
meta-agentينشئ agents جديدةWrite, WebFetch
docs-scraperيجلب ويلخص التوثيقWebFetch, Write, Read, Glob
code-reviewerيراجع جودة الكودRead, Grep, Glob
test-writerيولّد ملفات الاختبارRead, Write, Bash

٢ ما هي الـ AI Docs؟

هي ملفات معرفة متخصصة مخزنة في ai_docs/ — بتعطي الـ Agent فهماً عميقاً للنظام.

  • AGENTS.md → يخبر الـ Agent إيش المشروع وكيف يشغّله
  • AI Docs → يخبره كيف يشتغل النظام من الداخل ولماذا بُني هكذا

📁 الملفات النموذجية:

الملفالمحتوى
architecture.mdنظرة عامة، تدفق البيانات، القرارات التقنية
api-reference.mdكل endpoint موثق مع الـ request والـ response
component-guide.mdمكونات الـ design system مع الـ props والـ variants

⚠️ خطأ شائع

حشر محتوى AI Docs داخل AGENTS.md. الـ AGENTS.md يجب يكون 30-50 سطر فقط. اتركهم منفصلين.

٣ ما هي الـ Plan Commands؟

هي سير عمل قابلة لإعادة الاستخدام مخزنة في .pi/prompts/ — تُستدعى بـ / مثل /prime أو /quick-plan.

📋 شكل ملف الـ Command:

---
description: What this command does
argument-hint: <optional description>
---

# Command Name
Instructions for the agent to follow.
Use $ARGUMENTS to reference user input.

مثال: /quick-plan add a priority filter to the task list — كل شيء بعد اسم الـ command بيصير $ARGUMENTS

٤ خطوات التمرين العملي

١
أنشئ الـ meta-agent

أنشئ .pi/prompts/agents/meta-agent.md — مهمته الوحيدة إنشاء agents أخرى.

٢
أنشئ الـ docs-scraper agent

أنشئ .pi/prompts/agents/docs-scraper.md — يجلب التوثيق من URLs ويحفظ ملخصات منظمة.

٣
أنشئ AI Docs لـ TaskFlow

أنشئ 3 ملفات في ai_docs/: architecture.md، api-reference.md، component-guide.md

٤
أنشئ الـ Plan Commands

أنشئ 3 commands في .pi/prompts/: prime.md، quick-plan.md، load_ai_docs.md

٥
اختبر الـ Commands

جرّب /prime و /quick-plan add a search feature to TaskFlow — هل يعطي نتائج منظمة ومفيدة؟

🔍 التحقق من نسختك:

ls .pi/prompts/agents/   # meta-agent.md  docs-scraper.md
ls .pi/prompts/          # prime.md  quick-plan.md  load_ai_docs.md
ls ai_docs/              # architecture.md  api-reference.md  component-guide.md
head -5 .pi/prompts/agents/meta-agent.md  # frontmatter صحيح

٥ الدروس المستفادة

🧑‍💼Sub-agents = متخصصون — كل واحد يعمل شيء واحد بشكل ممتاز بأقل الأدوات.
📚AI Docs = قاعدة المعرفة — تُخبر الـ Agent كيف يشتغل النظام لا فقط كيف يشغّله.
⚡Commands = سير عمل جاهزة — تحوّل خطوات متعددة إلى أمر واحد ينتج نتائج ثابتة.
🔀معاً يُمكّنون العمل المتوازي — عدة agents تشتغل على أشياء مختلفة في نفس الوقت.
🗂️الفصل بين المسؤوليات — AGENTS.md للسياق، ai_docs للمعرفة، commands للسير، agents للتخصص.
💡 الفرق بين Level 1 و Level 2
Level 1: أعطيت الـ Agent وثيقة إحاطة 📄
Level 2: أعطيته فريق متخصص + قاعدة معرفة + إجراءات موحدة 🚀
