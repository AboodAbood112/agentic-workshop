🔧 Level 3: Skills, MCPs & Custom Tools

Class 1, Gear 3 — توسيع قدرات الـ Agent

"Sub-agents وSkills وMCPs كلهم بيعطوك نفس الشيء — أدوات مخصصة. الفرق فقط في طريقة التغليف."

١ ما هي الـ Skills؟

قدرات قابلة لإعادة الاستخدام معرّفة في .pi/skills/ — كل skill في مجلد خاص يحتوي على تعليمات + templates + أمثلة.

📁 بنية الملفات:

.pi/skills/
└── skill-name/
    ├── SKILL.md          # تعريف الـ skill
    └── templates/
        └── template.md  # قوالب جاهزة

🆚 الفرق بين Sub-Agent و Skill:

الخاصيةSub-AgentSkill
المكان.pi/prompts/agents/.pi/skills/
البنيةملف واحدمجلد مع templates
الاستدعاءتفويض تلقائيأمر / أو تفويض
السياقسياقه المستقليعمل ضمن السياق الحالي
النموذجيحدد نموذجهيستخدم نموذج الجلسة
💡 القاعدة العملية: إذا المهمة مستقلة → Sub-Agent. إذا تحتاج templates أو تعمل ضمن السياق الحالي → Skill.

⚠️ خطأ شائع

إنشاء Skill لما Sub-Agent كافي. لا تنشئ Skill إلا إذا عندك شيء حقيقي تحطه في المجلد.

٢ ما هي MCP Servers؟

Model Context Protocol — عمليات خارجية بتوفر أدوات تتجاوز الأدوات الأساسية الأربعة.

🛠️ أمثلة على MCPs:

الـ MCPالقدرة
Firecrawlتحويل أي URL لمحتوى منظم
Playwrightالتحكم بمتصفح حقيقي للاختبار
Database MCPالاستعلام المباشر من قواعد البيانات
Slack / GitHub / Jiraالاندماج مع أدوات العمل

⚙️ ملف الإعداد .mcp.json:

{
  "mcpServers": {
    "firecrawl": {
      "command": "npx",
      "args": ["-y", "firecrawl-mcp"],
      "env": { "FIRECRAWL_API_KEY": "your-key" }
    },
    "playwright": {
      "command": "npx",
      "args": ["-y", "@anthropic/mcp-playwright"],
      "env": {}
    }
  }
}

⚠️ خطأ شائع

حفظ API keys في الـ repo! ضع .mcp.json في .gitignore وشارك .mcp.json.sample بدون مفاتيح حقيقية.

٣ الأدوات المخصصة عبر Prime Prompts

أحياناً ما تحتاج Skill أو MCP — يكفي تصف النمط في AGENTS.md:

## Custom Tool: Research Pattern
When you need to research a library:
1. Search the web for the official docs
2. Fetch the getting-started page
3. Save a summary to ai_docs/<library-name>.md
4. Report what you learned

🔒 صلاحيات .pi/settings.json:

{
  "permissions": {
    "allow": [
      "Bash(bun run:*)",
      "Bash(git status)",
      "Bash(git diff:*)"
    ],
    "deny": [
      "Bash(rm -rf:*)",
      "Bash(git push:*)",
      "Bash(git reset --hard:*)"
    ]
  }
}

٤ خطوات التمرين العملي

١
أنشئ meta-agent skill

أنشئ .pi/skills/meta-agent/SKILL.md — نسخة Skill من meta-agent مع إشارة لـ template.

٢
أنشئ الـ Agent Template

أنشئ .pi/skills/meta-agent/templates/subagent-template.md بـ placeholders مثل <AGENT_NAME>، <TOOL_LIST>، <ROLE>.

٣
أنشئ MCP Config Sample

أنشئ .mcp.json.sample مع Firecrawl و Playwright — استخدم strings فارغة للـ API keys.

٤
أنشئ ملف الصلاحيات

أنشئ .pi/settings.json — اسمح لأوامر التطوير الآمنة، وامنع الأوامر المدمِّرة.

🔍 التحقق:

ls .pi/skills/meta-agent/           # SKILL.md  templates/
ls .pi/skills/meta-agent/templates/ # subagent-template.md
cat .mcp.json.sample | python3 -m json.tool
cat .pi/settings.json | python3 -m json.tool

٥ الدروس المستفادة

📐Skills = قدرات بقوالب — تضمن نفس البنية ونفس الجودة في كل مرة.
🌐MCPs = أدوات خارجية — تربط الـ Agent بـ APIs ومتصفحات وقواعد بيانات.
📝Custom Tools = تعليمات ذكية — صِف النمط في prime prompt والـ Agent يتبعه بلا إعداد.
🔒settings.json = شبكة الأمان — تحكم بما يُنفَّذ تلقائياً وما يحتاج موافقتك.
📈ابدأ بسيط — Custom tool أولاً، ثم Skill لما تحتاج templates، ثم MCP لما تحتاج وصولاً خارجياً.
💡 الثلاثة يخدمون نفس الهدف
Skills + MCPs + Custom Tools = توسيع الـ Core Four (Read, Write, Edit, Bash) 🚀
