📋 Level 5: Bug, Feature & Chore Templates

Class 1, Gear 5 — عمل مختلف = سير عمل مختلف

"إصلاح بق يبدأ بـ refactor هو إصلاح بق لن يُشحن أبداً."

١ المشكلة: Agent واحد لكل شيء

بدون templates، "أصلح هذا البق" و"ابنِ هذه الميزة" يعاملهما الـ Agent بنفس الطريقة — وهذا خطأ.

نوع العملالهدف الأساسيالمميّز الرئيسي
🐛 Bugإصلاح سلوك مكسورالاستنساخ أولاً، اختبار تراجع مطلوب
✨ Featureإضافة قدرة جديدةمدفوع بالـ Spec، خطط ثم ابنِ
🔧 Choreصيانة وتحسينتقييم أولاً، تحقق بعد التنفيذ
📐 Planتصميم المقاربةللقراءة فقط، ينتج ملف Spec

٢ تشريح الـ Template

frontmatterdescription + argument-hint → يجعل الأمر قابلاً للاكتشاف في القائمة
Variables$1 للمدخل الأول، $2 للمدخل الاختياري
Instructionsقيود صارمة على سلوك الـ Agent
Workflowخطوات مرقمة بالترتيب — تفصيلية وملزمة
Reportشكل الإخراج المنظم — ثابت في كل استدعاء

٣ الأنماط الأربعة

🐛 /bug — الاستنساخ أولاً
1. استنسخ 2. عزّل 3. أصلح 4. تحقق 5. اختبار تراجع

⚠️ قيد صارم: إصلاح أدنى فقط — لا refactoring للكود المحيط

✨ /feature — Spec أولاً
1. Spec 2. Plan 3. Build 4. Test 5. Review

🎛️ متغير SCOPE: "spec" | "plan" | "full"

🔧 /chore — التقييم أولاً
1. Assess 2. Execute 3. Validate 4. Document

⚠️ لا تغيير في السلوك الظاهر للمستخدم إلا إذا كان هذا هو الهدف

📐 /plan — للقراءة فقط
1. افهم المتطلبات 2. اقرأ السياق 3. اكتب Spec

🔒 قيد مطلق: لا تكتب كوداً، لا تعدّل ملفاً أبداً

📁 ينتج: specs/<name>.md

٤ خطوات التمرين العملي

١
أنشئ المجلد
mkdir -p workshop/app/.pi/prompts/
٢
أنشئ /bug template

أنشئ bug.md — 5 خطوات، قيد صارم: "لا تتجاوز خطوة الاستنساخ، وإذا فشلت أوقف وأبلّغ".

٣
أنشئ /feature template

أنشئ feature.md — متغيران: $1 الوصف، $2 الـ scope (افتراضي: "full"). الـ Spec يُحفظ في specs/.

٤
أنشئ /chore template

أنشئ chore.md — 4 خطوات. خطوة Validate تشغّل: bun run check + bun run lint + bun test.

٥
أنشئ /plan template

أنشئ plan.md — قراءة فقط. الـ Spec يشمل: Goal، Requirements، Technical Approach، Implementation Steps، Testing، Risks.

🧪 اختبار الـ Templates:

/bug "Tasks with status 'done' still appear in the 'To Do' filter"
/feature "Add task labels/tags with filtering" "spec"
/chore "Add input validation to all API endpoints"
/plan "Add task categories with color coding"

٥ الدروس المستفادة

🎯عمل مختلف = سير عمل مختلف — Bug يحتاج استنساخاً، Feature تحتاج Spec، Chore تحتاج تقييماً.
📏Templates تُشفّر أفضل الممارسات — كل استدعاء يتبع نفس المنهج الجيد، بغض النظر عن من يستدعيه.
🎛️المتغيرات تُضفي المرونة — $1 للمدخل، $2 للـ scope أو السياق الاختياري.
🔍Frontmatter يُسهّل الاكتشاف — الـ description يظهر في قائمة الأوامر عند كتابة /.
📊التقارير المنظمة تضمن الاتساق — كل استدعاء ينتج نفس شكل التقرير.
💡 ملخص الـ 5 Levels
L1: إحاطة الـ Agent 📄  |  L2: فريق + معرفة 👥  |  L3: توسيع القدرات 🔧
L4: حلقة مغلقة للتحقق 🔁  |  L5: سير عمل مخصص لكل نوع عمل 📋
