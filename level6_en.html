⛓️ Level 6: Prompt Chains & Agentic Workflows

Class 1, Gear 6 — Composable Primitives, Powerful Chains

"A single command is a tool. A chain of commands is a workflow. The difference is leverage."

1 What is Prompt Chaining?

A slash command invoking other commands in sequence, and passing data between them via files — instead of a massive command trying to do everything at once.

2 The Three Primitives

🔍 Scout
Input: Prompt + Scale
Output: List of .md files
Goal: Find relevant code
📐 Plan
Input: Prompt + File list
Output: Spec .md file
Goal: Design the approach
🔨 Build
Input: Spec file path
Output: Code + Report
Goal: Execute the plan
💡 Every Primitive works standalone or within a Chain — the Chain is an addition, not a requirement.

3 Passing Data Between Steps

The secret is passing the file path not its content — this keeps the context window clean.

Scout  →  agents/scout_files/relevant_files_XXX.md
                        ↓ (path)
Plan   →  specs/feature-name.md
                        ↓ (path)
Build  →  Code changes + Report

⚠️ Common Mistake

Building a massive command that combines Scout + Plan + Build in one prompt. The context window overflows, the Agent gets lost, quality degrades.

4 Sub-Agents in Parallel via Task Tool

Instead of one Agent searching sequentially — Scout spawns multiple Agents searching at the same time:

Agent 1
File Structure
Glob
Agent 2
Code Content
Grep
Agent 3
Dependencies
Import tracing

Results are merged, duplicates are removed, and written into a single file.

⚠️ Common Mistake

Creating Sub-Agents without sufficient context. Each Agent starts from scratch — make every agent prompt self-contained: Query + Strategy + Output Format.

5 Practical Exercise Steps

1
Create /scout command

Create scout.md — 2 variables: $1 prompt, $2 scale (default: 3). Spawns Agents in parallel, merges results into agents/scout_files/relevant_files_<timestamp>.md.

2
Create /plan_w_docs command

Create plan_w_docs.md — 3 variables: $1 prompt, $2 optional URLs, $3 Scout file path. Works standalone or within a Chain. Writes Spec to specs/.

3
Create /scout_plan_build orchestrator

Create scout_plan_build.md — calls all three sequentially via SlashCommand(). Important constraint: "Do not pause between steps — finish the whole workflow before stopping".

4
Create /build_in_parallel (Optional)

Create build_in_parallel.md — reads the Spec, categorizes steps: independent (parallel) vs sequential (await).

5
Test the full Chain
/scout "Add task categories with color coding"
/plan_w_docs "Add task categories" "" "agents/scout_files/relevant_files_*.md"
/scout_plan_build "Add task categories with color coding"

6 Key Takeaways

🧩Composable Primitives > Massive Commands — Small commands that work independently and compose into workflows.
📁Passing Paths not Content — keeps the context window clean between steps.
⚡Task Tool for Parallelism — Multiple Agents searching simultaneously instead of sequentially.
🔗Chains Automate Handoffs — Instead of 3 copy/pastes, one command finishes everything.
🔄Each Primitive is Independent — Chains are additive, not mandatory.
