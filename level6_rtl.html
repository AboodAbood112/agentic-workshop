١ ما هو Prompt Chaining؟

أمر slash يستدعي أوامر أخرى بالتسلسل، ويمرر البيانات بينها عبر ملفات — بدل أمر ضخم يحاول يعمل كل شيء دفعة واحدة.

/scout_plan_build
الـ Orchestrator
/scout
يبحث بالتوازي
←
relevant_files.md
←
/plan_w_docs
يصمم الـ Spec
←
spec.md
←
/build
ينفّذ بالتوازي

٢ الـ Primitives الثلاثة

🔍 Scout
المدخل: Prompt + Scale
المخرج: قائمة ملفات .md
الهدف: إيجاد الكود ذي الصلة
📐 Plan
المدخل: Prompt + قائمة ملفات
المخرج: ملف Spec .md
الهدف: تصميم المقاربة
🔨 Build
المدخل: مسار ملف Spec
المخرج: كود + تقرير
الهدف: تنفيذ الخطة
💡 كل Primitive يعمل منفرداً أو ضمن Chain — الـ Chain إضافة وليس إلزاماً.

٣ تمرير البيانات بين الخطوات

السر هو تمرير مسار الملف لا محتواه — هذا يُبقي نافذة السياق نظيفة.

Scout  →  agents/scout_files/relevant_files_XXX.md
                        ↓ (path)
Plan   →  specs/feature-name.md
                        ↓ (path)
Build  →  تغييرات الكود + تقرير

⚠️ خطأ شائع

بناء أمر ضخم يجمع Scout + Plan + Build في prompt واحد. نافذة السياق تمتلئ، الـ Agent يضيع، الجودة تنهار.

٤ Sub-Agents بالتوازي عبر Task Tool

بدل Agent واحد يبحث بالتسلسل — Scout ينشئ عدة Agents تبحث في نفس الوقت:

Agent 1
بنية الملفات
Glob
Agent 2
محتوى الكود
Grep
Agent 3
الـ Dependencies
Import tracing

النتائج تُدمج، التكرار يُحذف، وتُكتب في ملف واحد.

⚠️ خطأ شائع

إنشاء Sub-Agents بدون سياق كافٍ. كل Agent يبدأ من الصفر — اجعل prompt كل agent مكتفياً بذاته: الاستعلام + الاستراتيجية + شكل الإخراج.

٥ خطوات التمرين العملي

١
أنشئ /scout command

أنشئ scout.md — متغيران: $1 الـ prompt، $2 الـ scale (افتراضي: 3). ينشئ Agents بالتوازي، يدمج النتائج في agents/scout_files/relevant_files_<timestamp>.md.

٢
أنشئ /plan_w_docs command

أنشئ plan_w_docs.md — ثلاثة متغيرات: $1 الـ prompt، $2 URLs اختيارية، $3 مسار ملف Scout. يعمل standalone أو ضمن Chain. يكتب Spec في specs/.

٣
أنشئ /scout_plan_build orchestrator

أنشئ scout_plan_build.md — يستدعي الثلاثة بالتسلسل عبر SlashCommand(). قيد مهم: "لا تتوقف بين الخطوات — أكمل كل الـ workflow قبل التوقف".

٤
أنشئ /build_in_parallel (اختياري)

أنشئ build_in_parallel.md — يقرأ الـ Spec، يصنّف الخطوات: مستقلة (تتوازى) vs تسلسلية (تنتظر).

٥
اختبر الـ Chain كاملاً
/scout "Add task categories with color coding"
/plan_w_docs "Add task categories" "" "agents/scout_files/relevant_files_*.md"
/scout_plan_build "Add task categories with color coding"

٦ الدروس المستفادة

🧩Primitives قابلة للتركيب > أوامر ضخمة — أوامر صغيرة تعمل منفردة وتتركب في workflows.
📁تمرير المسارات لا المحتوى — يُبقي نافذة السياق نظيفة بين الخطوات.
⚡Task Tool للتوازي — Agents متعددة تبحث في نفس الوقت بدل التسلسل.
🔗الـ Chain تُؤتمت التسليم اليدوي — بدلاً من 3 نسخ/لصق، أمر واحد ينهي كل شيء.
🔄كل Primitive مستقل — الـ Chains إضافية، ليست إلزامية.
