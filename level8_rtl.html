🌐 Level 8: Out-Loop Systems & Webhooks

Class 2, Gear 1 — Agent يعمل بدونك

"اللحظة التي يستجيب فيها Agent للأحداث بدل الأوامر، تكون قد تجاوزت الخط من أداة إلى نظام."

١ التحول: In-Loop → Out-of-Loop

❌ In-Loop (Levels 1-7)
أنت → Agent → نتيجة → أنت تراجع

يتطلب حضورك — لا يعمل الساعة 3 صباحاً

✅ Out-of-Loop
حدث → Webhook → Agent → إشعار

يعمل تلقائياً — أنت المصمم لا المشغّل

🔥 أمثلة على Out-of-Loop:

  • PR يُفتح على GitHub → Agent يراجعه تلقائياً
  • رسالة Slack تصل → Agent يبدأ workflow بناء
  • تنبيه monitoring يطلق → Agent يحقق ويُبلّغ

⚠️ خطأ شائع

تجاهل خطوة Transform وإرسال raw payload للـ Agent مباشرة. البيانات الخام مليئة بالضجيج — Transform هو المكان الذي تُنظّف فيه السياق.

💡 مبدأ الفصل: الـ Webhook Server عام ومستقر. الـ Handler Scripts خاصة وتتطور. أضف workflows جديدة بإضافة handlers، لا بتعديل الـ Server.

٤ خطوات التمرين العملي

١
أنشئ Webhook Server

بناء HTTP server بـ Python يستقبل POST على /webhook، يُحلّل JSON، ويوجّه للـ handler الصحيح بناءً على event type.

curl -X POST http://localhost:8080/webhook \
  -d '{"event": "bug_report", "description": "Tasks not saving"}'
٢
أضف GitHub Integration

معالجة حدثين:

  • pull_request (opened) → workflow مراجعة كود
  • issues (bug label) → workflow تحقيق في بق

ملاحظة: GitHub يرسل نوع الحدث في header: X-GitHub-Event

٣
أضف Slack Integration

معالجة 3 slash commands:

  • /taskflow-build <desc> → workflow بناء
  • /taskflow-review → workflow مراجعة
  • /taskflow-status → حالة النظام

Slack يرسل form-encoded لا JSON — استخدم parse_qs(). استجب خلال 3 ثوانٍ.

٤
طبّق PITER الكامل لكل Handler

Execute Step:

subprocess.run(["pi", "-p", prompt], capture_output=True)

Transform Step: بناء markdown نظيف بالسياق المطلوب فقط.

٥
أضف Error Handling وLogging

كل handler في try/except. سجّل كل خطوة PITER مع التوقيت. أرسل تقرير فشل للمصدر حتى في حالة الخطأ — الفشل الصامت أسوأ أنواع الفشل.

٥ الدروس المستفادة

🎯Out-of-loop هو الهدف — Agent يحتاج حضورك لكل مهمة هو أداة. Agent يستجيب للأحداث تلقائياً هو نظام.
🏗️PITER يوفر البنية — 5 خطوات بالترتيب لكل workflow خارج الحلقة.
✨Transform هو مصدر القيمة — البيانات الخام مزعجة. Transform يُقدم للـ Agent بالضبط ما يحتاجه.
🔌فصل المسؤوليات — Server عام + Handlers خاصة = إضافة workflows بدون تعديل البنية.
🚨بلّغ عن الأخطاء دائماً — خطوة Report حيوية خاصةً للأخطاء. الأنظمة الآلية الفاشلة صامتاً هي الأسوأ.
💡 ملخص الـ 8 Levels
L1-L7: أنت في الحلقة — تكتب أمراً، الـ Agent ينفّذ
L8: أنت خارج الحلقة — الأحداث تُشغّل الـ Agent تلقائياً 🌐
