⚙️ Level 9: AI Developer Workflows (ADWs)

Class 2, Gear 2 — Deterministic code orchestrating non-deterministic Agents

"Deterministic code orchestrating non-deterministic Agents. The highest leverage point in Agentic programming."

1 The Hybrid Architecture

Pure Code
✅ Predictable and fast
✅ Testable
❌ Can't handle new problems
❌ Brittle to change
Pure Agent
✅ Creative and adaptive
✅ Handles ambiguity
❌ Loses context
❌ Skips steps
🏆 ADW (Hybrid)
✅ Predictable structure
✅ Creative execution
✅ Can't skip steps
✅ Complete audit trail

ADW Pattern:

Python (Deterministic) → The Skeleton — ensures step sequence
Model (Non-deterministic) → The Brain — thinks and writes code

The Skeleton guarantees the Brain won't skip or reorder any steps.

⚠️ Common Mistake

Trying to execute multi-step workflows using only prompt engineering. "First plan, then build, then review" in a single prompt is just a suggestion — the Agent will ignore it.

2 Step Lifecycle

log_step_start(step_name) # Always happens
│
├── Construct prompt for this step
├── subprocess.run(["pi", "-p", command])
├── Capture stdout/stderr + exit code
│
log_step_end(step_name, result) # Always happens

This gives you three things a Pure Agent lacks:

  • Full Observability — timestamped logs for the start and end of each step
  • Clear Boundaries — each step is a new model invocation, no context leakage
  • Retryability — you can retry just one failed step

3 Workflow Compositions

2 Steps: Plan→ Build For quick features
3 Steps: Plan→ Build→ Review With Quality Assurance
4 Steps: Plan→ Build→ Review→ Fix (If FAIL) Full automation with self-healing

🔀 Conditional Fix Pattern:

review_result = run_step("review")
if review_result.verdict == "FAIL":
    run_step("fix")
    run_step("review")  # Re-review after fixing
    # Maximum retry limit — usually 2-3

Python makes the conditional decision. The Model only executes within boundaries.

⚠️ Common Mistake

A review-fix-review loop without a maximum limit. Always set a retry limit — if the Agent fails to fix it in 3 attempts, it needs human intervention.

4 File Structure

adws/
  adw_modules/
    adw_runner.py          # The engine — executes steps with logging
    adw_logger.py          # Writes timestamped JSON logs
  adw_workflows/
    adw_plan_build.py           # 2 steps
    adw_plan_build_review.py    # 3 steps
    adw_plan_build_review_fix.py # 4 steps + condition
  adw_triggers/
    run_adw.py             # CLI trigger
    run_adw_background.py  # Background execution

5 Practical Exercise Steps

1
Create Logger Module

Create adw_logger.py — writes JSON logs for each step (timestamp + step name + event type + result). One JSON string per event line.

2
Create Runner Module

Create adw_runner.py — takes a list of steps, executes each step with subprocess.run(["pi", "-p", command]), and logs before and after.

3
Create 2-Step Workflow

Create adw_plan_build.py:

STEPS = [
    {"name": "plan", "command": "/plan", ...},
    {"name": "build", "command": "/build", ...}
]
runner.run(steps=STEPS, prompt=user_prompt)
4
Add Review → 3 steps

Create adw_plan_build_review.py — same as the 2-step + adds /review as a third step.

5
Add Conditional Fix → 4 steps

Create adw_plan_build_review_fix.py — if Review = FAIL, run Fix then review again. With a retry limit.

🧪 Testing:

python adws/adw_triggers/run_adw.py \
    --workflow plan_build \

    --working-dir /path/to/workshop/app

6 Key Takeaways

🤝Deterministic wraps non-deterministic — Python controls the flow, the Model controls the thinking. Neither can do the other's job well.
🚧Steps are boundaries, not suggestions — each step is a separate process invocation. It is impossible for the Agent to skip them.
👁️Observability is built-in — step lifecycle logging means you always know what happened and when.
🧩Composition = Extensibility — New workflows are just new step lists. Infrastructure is reused.
🐍Conditional Logic in Python — Run Fix? Python decides. Is execution creative? The Model decides.
